# バックアップ・復元手順

## 障害や誤削除に

最低限、次を一緒に保存します。

- `config.php`
- `reita.db`などのSQLiteデータベースファイル
- `img/`、`thumb/`、`thumbnail/`、`webp/`、`avif/`の各ディレクトリ
- 必要なら描画アニメーション・PSDなどの保存先

SQLiteは稼働中にDBファイルだけコピーすると、WAL使用時に最新データが含まれない可能性があります。そのため、管理モード停止中にコピーするか、SQLiteのバックアップ機能を使う手順が必要です。

## DBマイグレーション時の自動バックアップ

v3.5以降では、DBスキーマの更新前に`backup/`ディレクトリへ整合性の取れたSQLiteバックアップを自動作成します。ファイル名には更新前のスキーマバージョンと日時が入ります。

自動バックアップはDBだけが対象です。`config.php`や投稿画像は含まれないため、通常のバックアップの代わりにはなりません。復元する場合は稼働を止め、現在のDBを別の場所へ退避してから、バックアップファイルを`config.php`で指定したDB名へ戻してください。
